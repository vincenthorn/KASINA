* The **Go Direct Respiration Belt supports Bluetooth Low Energy (BLE)**, which can connect to web apps (like Vernier‚Äôs [Graphical Analysis Web App](https://graphicalanalysis.app.vernier.com/)) directly from Chrome.

### ‚ö†Ô∏è **Special Requirements & Limitations**

1. **Secure Context Only**:

   * Your app or page **must be served over HTTPS** (or be `localhost`) for Web Bluetooth to function.

2. **User Gesture Required**:

   * Bluetooth pairing can **only be triggered by a user action**, like a button click. This is a security requirement of the browser.

3. **BLE Permissions Dialog**:

   * Chrome will display a **device selection prompt** where the user must manually select the Respiration Belt from a list of BLE devices.

4. **Browser Support Varies by OS**:

   * **macOS:** Chrome supports Web Bluetooth well.
   * **iOS:** Web Bluetooth is not supported at all.
   * **Windows/Linux:** Generally supported, but driver issues can occasionally arise on Windows.

5. **Bluetooth Must Be Enabled**:

   * Bluetooth must be turned on in your macOS settings, and the sensor must be **awake and broadcasting** (usually by detecting that it‚Äôs being worn).

6. **No Background Connection Persistence**:

   * BLE connections made via Web Bluetooth do **not persist between page reloads**, so you‚Äôll need to reselect the device each time.
